## Decisions made + rationale

### 1. Abandoned the remote's `bwkit` design

The user said "let's abandon the remote." The remote at session start was a
mature `bwkit` Python package — an MCP gateway with CLI, metrics DB, and
`catalog.yaml` system. Fundamentally different architecture: remote treated
tools as pluggable adapters wrapping upstreams; new scaffold treats tools as
folders of vendored upstreams with their own READMEs.

The bwkit design is preserved only as tag `pre-reset-2026-04-23`. We did
NOT merge any of its files — a reconciliation would be a significant design
exercise that the user explicitly deferred to a later session.

**If the user wants to reconcile in the future:** `git checkout
pre-reset-2026-04-23` gets the old code back. The new layout could be
adapted to sit *underneath* the bwkit adapter model (each tool folder
exposing a `SPEC: ToolSpec`), but that's a non-trivial refactor.

### 2. codex-router: vendor + sidecar, not invasive fork

codex-lb (Soju06, v1.14.1) has **no provider abstraction** — it's hard-coded
to `chatgpt.com/backend-api` with ChatGPT's OAuth client ID baked in. The
user wants Vertex AI as a quota-exhaustion fallback. We rejected:

- **Vendor-as-is, defer Vertex** — loses the feature the user asked for.
- **Invasive fork** — creates a permanent merge burden every upstream release.

We chose a **sidecar proxy** (separate service sitting in front of codex-lb
at a different port) because it's a few hundred lines, leaves upstream
pristine, and can be rewritten without touching codex-lb. Design contract
lives in `codex-router/sidecar/README.md`. **No sidecar code was written
this session.**

### 3. markitdown: vendor `main` (0.1.6b2 beta), not stable v0.1.5

Upstream main has moved past the last tagged release (0.1.5, 2026-02-20)
to a `0.1.6b2` beta. We vendored main to stay current with bug fixes. If
stability becomes an issue, re-vendor from the `v0.1.5` tag — refresh steps
are in `markitdown/UPSTREAM.txt`.

### 4. `UPSTREAM.txt`, not `UPSTREAM`

Windows is case-insensitive. A bare `UPSTREAM` file collides with the
`upstream/` directory at the same level. The `.txt` suffix sidesteps the
collision without inventing a new convention.

### 5. yt-transcripts: stub only, backend TBD

User asked to add a bulk YouTube transcript extractor as the *next* tool
but chose "just add a stub" when asked for scope. The stub README fixes
the output contract (one Markdown per video with YAML frontmatter, aligned
with markitdown output) but leaves the extraction backend open.

### 6. Tag-then-force-push to protect remote history

Before force-pushing, we create an annotated tag `pre-reset-2026-04-23`
pointing at the old remote HEAD and push that tag first. This makes the
reset reversible via `git checkout pre-reset-2026-04-23` at any point.

## Risks / traps the next agent will hit

- **Python 3.13 hard requirement** for codex-lb native install. Upstream
  `pyproject.toml` pins `requires-python = ">=3.13"`. Windows users may
  not have 3.13 yet.
- **Port 1455 is immutable** for codex-lb. Upstream's `.env.example` warns
  *"Do not change the port. OpenAI dislikes changes."* It's the OAuth
  redirect URI registered with ChatGPT. If another service binds 1455,
  codex-lb's device-code login will fail silently.
- **Repo is ~37 MB** from the two vendored upstreams. Pushing a fresh
  repo this size is fine; just don't be surprised by the first push time.
- **`codex-router/upstream/` contains its own `AGENTS.md` and `CLAUDE.md`**
  from upstream. Those are ignored by the root `AGENTS.md` convention, but
  if a future agent reads them thinking they're repo-level, they'll be
  confused. Scope the root `AGENTS.md`'s authority explicitly if this
  becomes a problem.
- **Case-insensitive FS** (Windows). Sibling file + dir with the same base
  name will collide. Remember this any time you want to name a file
  `UPSTREAM`, `README`, `LICENSE` next to a directory of the same name.
- **No git history for the old bwkit design on `main`** after this push.
  Tag is the only path back. Don't delete it without user consent.
